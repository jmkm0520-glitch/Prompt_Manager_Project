# Prompt Manager Project

## 프로젝트 소개

Prompt Manager Project는 생성형 AI 프롬프트를 효율적으로 관리하기 위한 Python 콘솔 프로그램입니다.

생성형 AI를 활용하면서 작성한 다양한 프롬프트를 한곳에서 관리할 수 있도록 제작하였습니다.

사용자는 메뉴를 통해 프롬프트를 추가하고, 목록을 조회하며, 카테고리별 조회와 검색을 수행할 수 있습니다. 또한 자주 사용하는 프롬프트는 즐겨찾기로 등록하여 관리할 수 있습니다.

본 프로젝트는 Python 기초 문법과 Git/GitHub 버전 관리 학습을 목적으로 제작되었습니다.

---

# 개발 환경

- Python 3.14.6
- Visual Studio Code
- Git
- GitHub
- Windows

---

# 실행 방법

1. 프로젝트 폴더를 연다.

2. 터미널에서 아래 명령어를 입력한다.

```bash
python prompt_manager.py
```

3. 메뉴 번호를 입력하여 원하는 기능을 실행한다.

---

# 프로그램 기능

## 1. 프롬프트 추가

새로운 프롬프트를 등록합니다.

입력 항목

- 제목
- 내용
- 카테고리

기능

- 빈 값 입력 시 다시 입력하도록 구현
- 카테고리 선택
- 즐겨찾기 기본값 False
- 프로그램 실행 중 리스트에 저장

---

## 2. 프롬프트 목록

등록된 모든 프롬프트를 출력합니다.

출력 정보

- 번호
- 제목
- 카테고리
- 즐겨찾기 여부(⭐)

프롬프트가 없으면 안내 문구를 출력합니다.

---

## 3. 카테고리별 조회

카테고리를 선택하면 해당 카테고리의 프롬프트만 출력합니다.

카테고리에 프롬프트가 없으면 안내 문구를 출력합니다.

---

## 4. 프롬프트 검색

검색어를 입력하면

- 제목
- 내용

중 하나라도 포함하는 프롬프트를 검색합니다.

검색 결과가 없으면 안내 문구를 출력합니다.

---

## 5. 프롬프트 상세 보기

프롬프트 번호를 입력하면

- 제목
- 카테고리
- 즐겨찾기 여부
- 내용 전체

를 출력합니다.

잘못된 번호를 입력하면 다시 입력하도록 구현하였습니다.

---

## 6. 즐겨찾기 관리

프롬프트 번호를 입력하여

- 즐겨찾기 추가
- 즐겨찾기 해제

를 수행할 수 있습니다.

---

## 7. 즐겨찾기 목록

즐겨찾기로 등록된 프롬프트만 출력합니다.

즐겨찾기가 없으면 안내 문구를 출력합니다.

# 데이터 구조

프롬프트는 리스트 안에 딕셔너리를 저장하는 구조를 사용하였습니다.

```python
prompts = [
    {
        "title": "회의록 요약",
        "content": "...",
        "category": "텍스트 생성",
        "favorite": True
    }
]
```

### 리스트(List)를 사용한 이유

여러 개의 프롬프트를 순서대로 저장하고 반복문으로 쉽게 출력하기 위해 사용하였습니다.

### 딕셔너리(Dictionary)를 사용한 이유

하나의 프롬프트를

- 제목
- 내용
- 카테고리
- 즐겨찾기

형태로 관리하기 위해 사용하였습니다.

# 주요 함수 설명

| 함수 | 역할 |
|------|------|
| show_menu() | 메뉴 출력 |
| add_prompt() | 새로운 프롬프트 등록 |
| show_prompt_list() | 전체 프롬프트 목록 출력 |
| show_category() | 카테고리별 프롬프트 조회 |
| search_prompt() | 제목 또는 내용 검색 |
| show_prompt_detail() | 프롬프트 상세 정보 출력 |
| manage_favorite() | 즐겨찾기 추가 및 해제 |
| show_favorites() | 즐겨찾기 목록 출력 |
| main() | 프로그램을 반복 실행하고 메뉴를 제어 |

# 면접에서 설명할 수 있는 핵심 내용

### 왜 리스트를 사용했나요?

여러 개의 프롬프트를 순서대로 저장하고 반복문으로 쉽게 출력하기 위해 사용했습니다.

---

### 왜 딕셔너리를 사용했나요?

하나의 프롬프트를 제목, 내용, 카테고리, 즐겨찾기처럼 여러 속성으로 관리하기 위해 사용했습니다.

---

### 왜 while문을 사용했나요?

프로그램이 종료될 때까지 메뉴를 반복적으로 보여주기 위해 사용했습니다.

0을 입력하면 break로 반복을 종료하도록 구현했습니다.

---

### 왜 함수를 분리했나요?

각 기능을 독립적으로 관리하여 코드의 가독성을 높이고 유지보수를 쉽게 하기 위해 분리했습니다.

---

### 왜 Git을 사용했나요?

기능별 변경 이력을 관리하고 이전 버전으로 되돌릴 수 있으며 협업 시 코드 관리를 쉽게 하기 위해 사용했습니다.
